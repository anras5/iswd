Zdecydowalimy si na ocen par wariantw:

Wariant 10 > Wariant 17 - $ U(w_{10}) > U(w_{17}) $ - (wariant 17 ma ocen 1.00 na kryterium C3, a taki nie moe by lepszy od wariantu z przecitnymi ocenami jakim jest wariant 10)

Wariant 21 > Wariant 19 - $ U(w_{21}) > U(w_{19}) $ - (wariant 21 podobnie jak 18 ma warto 0 na kryterium C3 wic jest lepszy ni 19, mimo tego e 19 jest lepsze od 21 na kryterium C2)

# Utworzenie problemu UTA-GMS
problem = LpProblem("uta-gms", LpMaximize)

# Utworzenie zmiennej maksymalizowanej
epsilon = LpVariable("epsilon")

# Utworzenie zmiennych decyzyjnych dla kadego kryterium w problemie programowania matematycznego
decision_variables = {}
for column in df.columns:
    unique_values = df[column].sort_values().unique()
    decision_variables[column] = [LpVariable(f"x_{column}_{value}", 0, 1) for value in unique_values]

# Dodanie ograniczenia na normalizacj (maksymalna uyteczno globalna rwna 1)
# Poniewa wszystkie kryteria s typu koszt, moemy zaoy, e wariant o minimalnych wartociach jest najlepszy
problem += lpSum([x[0] for x in decision_variables.values()]) == 1

# Dodanie ograniczenia na normalizacj (minimalna uyteczno globalna rwna 0)
# Poniewa wszystkie kryteria s typu koszt, moemy zaoy, e wariant o maksymalnych wartociach jest najgorszy
problem += lpSum([x[-1] for x in decision_variables.values()]) == 0

# Dodanie ogranicze na monotoniczno funkcji czstkowych
# W problemie uywana jest funkcja oglna, wic nie ma potrzeby wyliczania interpolacji liniowej
# Wystarczy doda ograniczenia, ktre gwarantuj, e kolejne zmienne decyzyjne maj nierosnce wartoci uytecznoci
for column in df.columns:
    for i in range(len(decision_variables[column])-1):
        problem += decision_variables[column][i] >= decision_variables[column][i+1]

# Dodanie ogranicze zwizanych z preferencjami decydenta
for preference in PREFERENCES:
    alt_1 = df.loc[preference[0]].to_dict()
    alt_1_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v in alt_1.items()]
        
    alt_2 = df.loc[preference[1]].to_dict()
    alt_2_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v, in alt_2.items()]

    problem += lpSum(alt_1_variables) >= lpSum(alt_2_variables) + epsilon

problem += epsilon
problem.solve(solver=GLPK(msg=False))
solution = {variable.name: variable.varValue for variable in problem.variables()}

Analiza moliwych i koniecznych relacji preferencji za pomoc metody UTA-GMS

W ramach zadania 2 naley dla kadej pary wariantw przeprowadzi badanie:

  1. Czy istnieje midzy nimi konieczna relacja preferencji
  2. Jeeli relacja konieczna nie zachodzi, to czy istnieje relacje moliwa

Po uzyskaniu wynikw moliwe bdzie wywietlenie diagramu przewyszania.

Budowanie problemu programowania matematycznego odbywa si na podobnej zasadzie co w podstawowej metodzie UTA. Jedyn rnic jest dodanie odpowiedniego ograniczenia badajcego relacj konieczn i moliw. Dodanie odpowiednich ogranicze zostao oznaczone w kodzie programu.

In[9]:
%%time
NECESSARY = 1
POSSIBLE = 2

alternatives = df.index
df_relations = pd.DataFrame(0, index=alternatives, columns=alternatives)
for alt_1_id in alternatives:
    for alt_2_id in alternatives:
        if alt_1_id == alt_2_id:
            df_relations.loc[alt_1_id, alt_2_id] = NECESSARY
            continue

        # Badanie relacji koniecznej alt_1 > alt_2
        problem = LpProblem("uta-gms", LpMaximize)
        epsilon = LpVariable("epsilon")
        decision_variables = {}
        for column in df.columns:
            unique_values = df[column].sort_values().unique()
            decision_variables[column] = [LpVariable(f"x_{column}_{value}", 0, 1) for value in unique_values]
        problem += lpSum([x[0] for x in decision_variables.values()]) == 1
        problem += lpSum([x[-1] for x in decision_variables.values()]) == 0
        for column in df.columns:
            for i in range(len(decision_variables[column])-1):
                problem += decision_variables[column][i] >= decision_variables[column][i+1]
        for preference in PREFERENCES:
            alt_1 = df.loc[preference[0]].to_dict()
            alt_1_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v in alt_1.items()]
            alt_2 = df.loc[preference[1]].to_dict()
            alt_2_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v, in alt_2.items()]
            problem += lpSum(alt_1_variables) >= lpSum(alt_2_variables) + epsilon
        # Dodanie preferencji badajcej relacj konieczn
        alt_1 = df.loc[alt_1_id].to_dict()
        alt_1_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v in alt_1.items()]
        alt_2 = df.loc[alt_2_id].to_dict()
        alt_2_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v, in alt_2.items()]
        problem += lpSum(alt_2_variables) >= lpSum(alt_1_variables) + epsilon
        problem += epsilon
        problem.solve(solver=GLPK(msg=False))
        solution = {variable.name: variable.varValue for variable in problem.variables()}
        epsilon_value = solution['epsilon']
        if epsilon_value <= 0:
            df_relations.loc[alt_1_id, alt_2_id] = NECESSARY
            continue
        
        # Badanie relacji moliwej alt_1 > alt_2
        problem = LpProblem("uta-gms", LpMaximize)
        epsilon = LpVariable("epsilon")
        decision_variables = {}
        for column in df.columns:
            unique_values = df[column].sort_values().unique()
            decision_variables[column] = [LpVariable(f"x_{column}_{value}", 0, 1) for value in unique_values]
        problem += lpSum([x[0] for x in decision_variables.values()]) == 1
        problem += lpSum([x[-1] for x in decision_variables.values()]) == 0
        for column in df.columns:
            for i in range(len(decision_variables[column])-1):
                problem += decision_variables[column][i] >= decision_variables[column][i+1]
        for preference in PREFERENCES:
            alt_1 = df.loc[preference[0]].to_dict()
            alt_1_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v in alt_1.items()]
            alt_2 = df.loc[preference[1]].to_dict()
            alt_2_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v, in alt_2.items()]
            problem += lpSum(alt_1_variables) >= lpSum(alt_2_variables) + epsilon
        # Dodanie preferencji badajcej relacj moliw
        alt_1 = df.loc[alt_1_id].to_dict()
        alt_1_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v in alt_1.items()]
        alt_2 = df.loc[alt_2_id].to_dict()
        alt_2_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v, in alt_2.items()]
        problem += lpSum(alt_1_variables) >= lpSum(alt_2_variables)
        problem += epsilon
        problem.solve(solver=GLPK(msg=False))
        solution = {variable.name: variable.varValue for variable in problem.variables()}
        epsilon_value = solution['epsilon']
        if epsilon_value > 0:
            df_relations.loc[alt_1_id, alt_2_id] = POSSIBLE

Reprezentatywna funkcja uytecznoci

W ramach zadania 3 naley skonstruowa reprezentatywn funkcj uytecznoci. Korzysta ona z wynikw otrzymanych w metodzie $UTA^{GMS}$. Poniej przedstawiony zosta kod programu, ktry modeluje problem programowania matematycznego. Dodatkowymi ograniczeniami w porwnaniu do standardowej metody UTA s ograniczenia wynikajce z relacji koniecznych i moliwych. Problem zosta obliczony w jednym kroku dziki zastosowaniu w funkcji celu wyraenia M*epsilon - delta, gdzie M to dua liczba - w tym przypadku ustawiona na warto 100.

M = 100
epsilon = LpVariable("epsilon")
delta = LpVariable("delta")
decision_variables = {}
for column in df.columns:

    decision_variables[column] = [LpVariable(f"x_{column}_{value}", 0, 1) for value in unique_values]
problem += lpSum([x[0] for x in decision_variables.values()]) == 1
problem += lpSum([x[-1] for x in decision_variables.values()]) == 0
for column in df.columns:
    for i in range(len(decision_variables[column])-1):
        problem += decision_variables[column][i] >= decision_variables[column][i+1]
for preference in PREFERENCES:
    alt_1 = df.loc[preference[0]].to_dict()
    alt_1_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v in alt_1.items()]
    alt_2 = df.loc[preference[1]].to_dict()
    alt_2_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v, in alt_2.items()]
    problem += lpSum(alt_1_variables) >= lpSum(alt_2_variables) + epsilon

# Dodanie preferencji wynikajcych z relacji koniecznych i moliwych
for alt_1_id in df_relations.index:
    for alt_2_id in df_relations.columns:
        if alt_1_id == alt_2_id:
            continue
        relation = df_relations.loc[alt_1_id, alt_2_id]
        if relation == NECESSARY:
            alt_1 = df.loc[alt_1_id].to_dict()
            alt_1_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v in alt_1.items()]
            alt_2 = df.loc[alt_2_id].to_dict()
            alt_2_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v, in alt_2.items()]
            problem += lpSum(alt_1_variables) >= lpSum(alt_2_variables) + epsilon
        else:
            alt_1 = df.loc[alt_1_id].to_dict()
            alt_1_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v in alt_1.items()]
            alt_2 = df.loc[alt_2_id].to_dict()
            alt_2_variables = [next((variable for variable in decision_variables[k] if str(variable) == f"x_{k}_{v}"), None) for k, v, in alt_2.items()]
            problem += lpSum(alt_1_variables) - lpSum(alt_2_variables) <= delta

problem += M*epsilon-delta

Wnioski

W ramach projektu przeprowadzono badania na zbiorze danych "Nuclear waste management" przy uyciu metod UTA oraz UTA-GMS. Badania wykazay, e:

  1. Metoda UTA wraz z uyciem funkcji oglnej uytecznoci czstkowej bardzo atwo potrafi dopasowa si do preferencji decydenta. Dla omawianego problemu istotne i brane pod uwag w wyliczaniu rankingu w etapie agregracji s trzy kryteria - C1, C2 i C3, a kryterium C4 zostao wyzerowane, co moe by spowodowane brakiem ogranicze na wagi kryteriw. Wynik podzieli cay zbir danych jedynie na rne 3 rangi.

  2. W wyniku dziaania metody UTA-GMS pozyskane zostay relacje konieczne i moliwe przedstawione przy pomocy diagramu Hasse. Mona zaobserwowa, e zbir danych zosta podzielony na 4 rangi, natomiast trudno powiedzie na podstawie grafu, ktry z wariantw jest faktycznie najlepszy.

  3. Funkcja reprezentatywna otrzymana przy pomocy relacji koniecznych i moliwych uzyskanych z metody UTA-GMS dostarczya bardziej zrnicowanych wynikw. Jest to spowodowane wiksz liczb ogranicze dodanych do problemu programowania matematycznego. Warianty uzyskuj uytecznoci ze zbioru 5 rnych wartoci. Najlepszy okaza si wariant 21. Jest to spowodowane dostarczonymi do metody preferencjami. Wariant 21 jest lepszy od wariantu 19, ktry z kolei jest lepszy od wariantu 16. Powoduje to, e wariant 21 jest mocno premiowany.

Dalsze badania mogyby obejmowa zastosowanie wikszej iloci porwna parami, wprowadzenie porwna reprezentujcych nierozrnialno pomidzy wariantami, czy te rozszerzenia takie jak GRIP lub moliwe pozycje najlepsze/najgorsze dla danego wariantu. W ramach dalszej analizy mona byoby rwnie skorzysta z samplera, ktry dostarczyby informacje na temat stochastycznej analizy regresji porzdkowej i indeksw akceptowalnoci dla wariantw.
